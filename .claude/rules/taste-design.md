# Taste Design Rule — Always Apply for UI/UX Work

When doing ANY UI/UX design, frontend component creation, styling, or visual work:

## Step 0: Read Design References (MANDATORY — before any code)
- Read the relevant DESIGN.md files in `.claude/design-references/` FIRST
- These are enterprise SaaS design systems (Linear, Stripe, Supabase, Superhuman)
- Extract applicable patterns: spacing, typography hierarchy, card depth, animation restraint
- Adapt patterns to VroomX brand (orange #fb7232, warm off-white #f7f7f5, widget-card system)
- This step is NOT optional — even for "small" changes like padding, color, or icons

## Step 1: Activate Skills
1. `design-taste-frontend` (taste baseline — always first)
2. `frontend-design` (project-specific design rules)
3. `ui-ux-pro-max` (comprehensive design intelligence)
4. Any other relevant skills (senior-frontend, react-best-practices, etc.)

## Step 2: Cross-Reference
- Cross-reference taste-skill principles with existing project design tokens (globals.css vars, brand colors, glass effects)
- Apply taste-skill's bias correction: no generic layouts, no AI design clichés, no Inter font, no neon/purple defaults
- Use taste-skill's creative arsenal for premium interactions when building new components
- Respect the project's existing design system (Tailwind v4 + shadcn/ui + globals.css tokens)

## Step 3: Implement
- Match the enterprise quality seen in the DESIGN.md references
- Every visual change should make VroomX feel more like Linear/Stripe — not less
